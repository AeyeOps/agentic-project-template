# Product Docs

Requirements, specifications, and user stories.

## Conventions

- Clear scope and success criteria for each PRD
- Link to `architecture/` for technical implementation details
- Version in filename when superseding: `auth-v2.md`
- Move previous versions to `archive/` with superseded note

## Content Types

- Product Requirements Documents (PRDs)
- Feature specifications
- User stories and acceptance criteria
- API specifications (consumer-facing)

## Lifecycle

1. `roadmap/` item prioritized for next cycle
2. PRD created here with requirements
3. `architecture/` updated with technical design
4. Previous PRD version archived when superseded
